Lunity Player (parity)

Same bootstrap as mix lunity.player: welcome → hello → auth → join (hints only) → assigned. Then subscribe_state → actions (parity). Pass token and jwt in the query string, or use the dev form below to mint a JWT (same-origin POST).

Example: /player?token=…&jwt=… · optional hints=<JSON> · auth_only=1 stops after ack · skip_followup=1 stops after assigned (no subscribe/actions) · resume=1 sends auth with resume: true after a disconnect (same JWT; matches mix lunity.player --resume) · live=1 keeps the socket open and renders periodic state.ecs below (same contract as mix lunity.player_window).
